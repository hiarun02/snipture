# Contributing to Snippify

Thanks for your interest in contributing! This guide explains how to propose changes, report issues, and submit pull requests.

## Ways to Contribute

- Report bugs and suggest enhancements
- Improve documentation
- Fix issues or add features
- Review pull requests

## Reporting Issues

1. Search existing issues to avoid duplicates.
2. Provide a clear title and description.
3. Include steps to reproduce, expected behavior, and actual behavior.
4. Add screenshots or logs if relevant.

## Development Setup

1. Install dependencies:
   - npm: `npm install`
   - pnpm: `pnpm install`
2. Run the dev server:
   - npm: `npm run dev`
   - pnpm: `pnpm dev`

## Pull Request Guidelines

- Keep changes focused and small when possible.
- Follow existing code style and conventions.
- Add or update tests when applicable.
- Update documentation if behavior changes.
- Ensure `npm run lint` passes.

## Commit Messages

Use short, descriptive commit messages (e.g., "Fix editor export sizing").

## Code Review

Maintainers will review PRs for quality, correctness, and alignment with project goals. You may be asked to make changes.
